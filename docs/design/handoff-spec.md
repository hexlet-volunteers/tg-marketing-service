# Handoff: tgpulse — аналитика Telegram-каналов с AI

## Overview
Веб-приложение аналитики Telegram-каналов (аналог tgstats) с AI-слоем: статистика каналов и групп, каталог/поиск, сравнение, кабинет автора с рекомендациями «о чём писать», подборки, блог, тарифы, личный кабинет и админка с модерацией пользовательских заявок.

Прототип — единый SPA с клиентской навигацией между экранами и модальными окнами.

## About the Design Files
Файлы в этом пакете — **дизайн-референс, сделанный в HTML** (один Design Component `Telegram Аналитика.dc.html`). Это прототип, показывающий задуманный вид и поведение, **а не продакшн-код для копирования**.

Задача — **пересоздать эти экраны в вашем реальном фронтенде** (React 19 + Vite, который уже есть в репозитории) на **реальной библиотеке Mantine** и вашем стейт-менеджере (Redux Toolkit / React Router уже подключены). Инлайн-стили из прототипа переносить не нужно — используйте компоненты `@mantine/core` и тему Mantine.

## Fidelity
**High-fidelity (hifi).** Финальные цвета, типографика, отступы, состояния и интерактив. Верстку следует воспроизвести близко к пикселю, но **штатными компонентами Mantine**, а не копированием инлайн-стилей.

⚠️ Важно про «Mantine»: прототип написан в **визуальном стиле Mantine на инлайн-CSS**, не на самой библиотеке. В реальном коде замените ручную верстку на компоненты Mantine (см. маппинг ниже). Значения цветов/радиусов/типографики ниже подобраны под дефолтную тему Mantine, чтобы совпало.

## Соответствие: прототип → компоненты Mantine
- Кнопки (`.btnp`, залитые/outline) → `<Button>` (`variant="filled" | "light" | "outline" | "subtle"`, `color="tgblue"`).
- Карточки (белые блоки с бордером и радиусом) → `<Card withBorder radius="md">` / `<Paper>`.
- Модалки (профиль, апгрейд, добавить канал, конкуренты, композер, вход/регистрация/сброс, редактор статьи/подборки) → `<Modal>` (+ `<Modal.Root>` для кастомной шапки).
- Сегментные переключатели (7д/30д/90д, Все/Каналы/Группы, помесячно/год, табы админки) → `<SegmentedControl>`.
- Табы админки можно и на `<Tabs>`.
- Таблицы (посты, сравнение, админ-каналы, очередь модерации) → `<Table>`.
- Селекты (категория канала, категория заявки) → `<Select>` (данные из справочника категорий).
- Тумблеры (уведомления, верификация) → `<Switch>`.
- Поля ввода/поиск → `<TextInput>` (с `leftSection` для иконки), `<Textarea>` для композера/статьи.
- Чипы категорий/типов/бейджи → `<Badge>` / `<Chip>`.
- Прогресс тарифа → `<Progress>`.
- Аватары каналов (инициалы на градиенте) → `<Avatar>` (можно с `variant="gradient"`).
- Сайдбар-навигация → `<NavLink>` внутри `<AppShell.Navbar>`; топбар → `<AppShell.Header>`.
- Тепловая карта лучшего времени — кастом (грид `<Box>` с фоновыми цветами шкалы).
- Графики (рост, охваты) — в прототипе это inline-SVG. В проде возьмите charting-библиотеку (`@mantine/charts` на базе Recharts — `AreaChart`, `BarChart`).
- Иконки — в прототипе эмодзи/инлайн-SVG. Замените на icon-набор проекта (в репозитории есть `@heroicons/react` и `lucide-react`).

## Interactions & Behavior
- **Навигация**: клик по пункту сайдбара / кнопкам меняет активный экран (в проде — React Router routes).
- **Поиск в топбаре**: ввод фильтрует каталог по title/username/категории; если был на лендинге/в кабинете — переключает на каталог.
- **Каталог**: тип (Все/Каналы/Группы) и категория фильтруют список; сортировка по подписчикам desc.
- **Сравнение**: тоггл каналов (макс 3); «лучший» вычисляется по каждой метрике; AI-вердикт — производный текст.
- **Конкуренты**: добавление по @username/ссылке — если найден в базе, фильтруется для выбора; если нет — «Добавить в базу» (создаёт канал + заявку на модерацию, сразу доступен для сравнения). «Сравнить все» → экран сравнения с [свой канал, ...конкуренты] (первые 3).
- **Модерация**: одобрение с категорией → публикация в каталог с дедупом по @username и `verified=true`; отклонение → удаление из очереди.
- **Апгрейд тарифа**: 3 шага — тарифы (помесячно/год, −20% меняет цены) → оплата (карта/срок/CVC/email, «Оплатить N ₽») → успех; после успеха тариф пользователя меняется, баннер «Перейдите на Pro» скрывается, сайдбар показывает новый тариф.
- **Композер**: «Написать» открывает модалку с AI-черновиком по теме идеи (редактируемый `Textarea`), прогноз охвата и лучшее время, «Запланировать»/«Опубликовать».
- **Добавить канал/группу**: тип (Канал/Группа) + @username/ссылка → добавляет в «Мои каналы», создаёт заявку на модерацию, переводит в AI-кабинет.
- **Модалки**: клик по оверлею закрывает (stopPropagation на карточке); крестик закрывает.
- **Hover**: карточки каналов приподнимаются (translateY -2, тень); кнопки — brightness; строки таблиц — светлый фон.

## State Management
Ключевой стейт (в прототипе — один класс; в проде разложить по Redux-слайсам / роутеру):
- `screen` — активный экран (в проде: маршрут).
- `q` — поисковый запрос топбара; `catalogType` (all/channel/group); `cat` (активная категория).
- `compare` — массив id (до 3) для экрана сравнения; `competitors` — id конкурентов в кабинете; `compQuery` — поиск конкурента; `extraChannels` — добавленные пользователем каналы (вне сид-базы).
- `modal` — какой модал открыт (login/register/recovery/editProfile/upgrade/addChannel/addCompetitor/composer/article/collectionEd/null).
- `plan` — тариф пользователя; `upgradeStep` (plans/pay/done); `selectedPlan`; `upgradeAnnual`.
- `myChannels` — каналы пользователя; `moderationQueue` — очередь заявок; `chOverrides` — правки категории/верификации в админке; `customPosts` / `customCollections` — созданные в админке; `artDraft` / `collDraft` — черновики редакторов; `adminTab`.
- `collection` / `blogSlug` / `legalTab` — выбранная подборка/статья/вкладка правового.
- Данные (каналы, группы, посты блога, подборки, инсайты) — статические справочники; в проде — API (Django-бэкенд уже отдаёт `get_data()` для каналов/групп).
